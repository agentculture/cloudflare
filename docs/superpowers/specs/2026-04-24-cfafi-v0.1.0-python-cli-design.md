# cfafi v0.1.0 вЂ” Python CLI on PyPI, Bash Skills Thinned

## Context

cfafi today is a pair of bash-based skills (`cloudflare/`, `cloudflare-write/`) under `.claude/skills/` plus supporting bats tests. Every CloudFlare verb is a standalone `cf-*.sh` invoked by SKILL.md or by hand. That worked to establish the shape (markdown-default / `--json` flag, dry-run / `--apply`, pagination via `cf_api_paginated`, `.env`-loading via `_lib.sh`), but it leaves the interface untyped, under-tested relative to Python tooling, and hostile to reuse outside this repo.

**Goal:** ship a first Python package named **`cfafi`** to PyPI that exposes the same surface as `cfafi <noun> <verb>` вЂ” a walking skeleton in v0.1.0 вЂ” and mirrors the architecture, CI, and release cadence of **`afi-cli`** (sibling repo `/home/spark/git/afi-cli`, package `afi-cli` on PyPI, command `afi`). "Code locked with code": the Python implementation becomes the authoritative logic with pytest coverage and type discipline; the bash scripts and bats tests migrate verb-by-verb over subsequent PRs. Skills become thinner вЂ” their SKILL.md points at `cfafi вЂ¦` commands, not at bash wrappers.

This PR is intentionally the **pipeline PR**, not the big-port PR. It proves end-to-end: uv-managed package в†’ GitHub Actions в†’ TestPyPI on PR, PyPI on merge в†’ installed `cfafi` command в†’ credentials from environment в†’ first 5 verbs working. Every subsequent verb is a small, mechanical follow-up PR with a patch version bump.

A separate follow-up PR handles culture mesh onboarding (agent config init, join `spark`, participate in `#general`). This PR only lifts the CLAUDE.md constraint that forbids joining.

## Outcome after this PR

- `uv tool install cfafi` works from PyPI.
- `cfafi whoami`, `cfafi zones list`, `cfafi dns create <zone> <type> <name> <content> [--apply]`, `cfafi learn`, `cfafi explain <noun> [<verb>]` all work end-to-end against live CloudFlare when `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` are exported.
- CI: PR runs tests + bats + markdownlint + TestPyPI publish; merge to main publishes to PyPI via OIDC Trusted Publishing.
- Every PR from now on must bump the version in `pyproject.toml` and prepend a `CHANGELOG.md` entry (enforced by a `version-check` job).
- Bash scripts still work exactly as before вЂ” no regressions for anyone using the existing skills.
- CLAUDE.md's "do not join mesh" line is removed; skill dirs renamed to `cfafi/` + `cfafi-write/`.

## Architecture

### CLI shape вЂ” nested noun/verb (locked)

```text
cfafi whoami                                       # auth check, markdown KV
cfafi status                                       # (later) digest
cfafi learn                                        # agent self-teaching prompt
cfafi explain <noun> [<verb>]                      # markdown docs per verb

cfafi zones list                                   # markdown table
cfafi dns list <zone>                              # (later)
cfafi dns create <zone> <type> <name> <content> [--apply]
cfafi dns delete вЂ¦                                 # (later)

cfafi workers scripts list                         # (later)
cfafi workers scripts create вЂ¦                     # (later)
cfafi workers scripts delete вЂ¦                     # (later)
cfafi workers routes list [--zone <zone>]          # (later)
cfafi workers routes create <zone> <pattern> <script>  # (later)
cfafi workers routes delete вЂ¦                      # (later)

cfafi pages projects list                          # (later)
cfafi pages projects create <project> --github <repo>  # (later)
cfafi pages projects delete вЂ¦                      # (later)
cfafi pages deployments list <project>             # (later)
cfafi pages deployments delete <project> <id>      # (later)
cfafi pages deployments purge <project>            # (later)

cfafi redirects create <zone> <from> <to>          # (later)
```

Global flags on every verb: `--json` (raw CloudFlare envelope or synthetic envelope for composites). Mutations additionally take `--apply` (dry-run by default; prints the request that would be sent).

### Runtime deps вЂ” stdlib only (locked)

`dependencies = []` in pyproject. HTTP via `urllib.request`. JSON via `json`. Pagination is our own loop over the `result_info.total_pages` pattern (port of `cf_api_paginated` from `.claude/skills/cloudflare/scripts/_lib.sh`). This is the single AgentCulture house-style trait most worth keeping вЂ” `afi-cli` enforces the same.

### Credentials вЂ” env vars only, no filesystem snooping (locked)

The installed CLI reads `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` from the environment. It does **not** walk up directories looking for `.env`, does not consult `~/.config/cfafi/`, does not touch the keychain. If a var is missing, it errors with a pointer to `cfafi learn` (which explains secure loading patterns) and to `docs/SETUP.md`.

Callers are responsible for secure delivery:

```bash
# Agent-script recipe (documented in `cfafi learn`):
set -a; . /path/to/agent-owned/.env; set +a
cfafi zones list
```

The recommended secure pattern (in `cfafi learn`): store credentials in a file owned by the agent's POSIX user with mode `0600`, outside any git-tracked directory. Future work can add an opt-in `~/.config/cfafi/config` with profiles (mode-checked on read) as a strict superset.

During coexistence, `.claude/skills/cfafi/scripts/_lib.sh` continues to load the repo-root `.env` so bash scripts run as they always have вЂ” the Python CLI simply ignores `.env`.

### Patterns to port from `.claude/skills/cloudflare/scripts/_lib.sh`

- `cf_api` в†’ `cfafi._api.http_request(method, path, *, payload=None, query=None)`. Reads `CLOUDFLARE_API_TOKEN` via `_env.require_env`, sends `Authorization: Bearer вЂ¦`, parses JSON, raises `CfafiError(EXIT_API)` on non-2xx with the CloudFlare error's `code` + `message`.
- `cf_api_paginated` в†’ `cfafi._api.paginate(path, *, query=None)` generator yielding result rows; walks `result_info.total_pages`. URL-encodes `page` and merges into `query`.
- `.env` loader вЂ” **not ported**. Intentional.
- Credential-missing error message вЂ” ported verbatim, with `cfafi learn` and `docs/SETUP.md` pointers.

### Output conventions

- Default: **markdown**. Tables (pipe-delimited with `| --- |` separator row) for lists; key-value (`- **key:** value`) for single objects. Matches existing bash output byte-for-byte where feasible so agents that parse the output don't regress.
- `--json` anywhere: raw CloudFlare envelope passthrough. For composite commands (future `cfafi status`), a synthetic `{ "result": { вЂ¦ }, "success": true, "errors": [], "messages": [] }` envelope.
- Errors go to stderr in the afi-cli structured format: two lines `error: <message>\nhint: <remediation>\n` (the `hint:` line is omitted when remediation is empty). With `--json`: `{ "code": N, "message": "вЂ¦", "remediation": "вЂ¦" }` to **stderr** вЂ” not stdout вЂ” so the stdout/stderr split is preserved in JSON mode too. Exit code is nonzero and comes from the `CfafiError.code` field; see `cfafi/cli/_errors.py` for the policy (0 success, 1 user error, 2 env error, 3 auth, 4 upstream API). Implementation: `cfafi/cli/_output.py:emit_error`.

### Write safety (inherits bash semantics)

Mutations default to **dry-run**: print the method/path/payload they *would* send, then exit 0. `--apply` actually commits. Exit code 0 for "dry-run printed," 0 for "applied successfully," nonzero for errors. Tests assert both paths вЂ” `--apply` path uses a fixture stub that records the request.

## Testing strategy

- **Pytest** for all Python code. Parallel via `pytest-xdist`, coverage gate `--cov=cfafi --cov-report=term --fail-under=70` (matching afi-cli).
- `tests/conftest.py` installs a fixture loader that monkeypatches `cfafi._api.http_request` to return JSON loaded from `tests/fixtures/<name>.json`. Same fixture files the bats suite already uses вЂ” zero duplication on fixture data.
- Tests cover: each CLI verb (markdown + `--json` both), pagination walk across a 3-page fixture, dry-run vs `--apply` for `dns create`, URL-encoding of `zone` and `name` args, missing-env-var error path, 401 auth-error path, 4xx CloudFlare-error path.
- **Bats suite continues to run** in CI (`tests/bats/`, `tests/shellcheck.sh`) until bash scripts are ported. Every PR runs both.
- **Markdownlint** via `scripts/lint-md.sh` (vendored from afi-cli); pre-commit hook optional.
- **Version-check CI job**: fails the PR if `pyproject.toml` version equals main's version, or if `CHANGELOG.md` has no new entry.

## PyPI publishing pipeline

Copy afi-cli's `.github/workflows/publish.yml` with three adjustments:

1. Package name `afi-cli` в†’ `cfafi` (also the project name on PyPI). **Pre-flight:** before merging this PR, manually check `https://pypi.org/project/cfafi/` and `https://test.pypi.org/project/cfafi/` вЂ” both must be either free or owned by the AgentCulture org. TestPyPI and PyPI are separate namespaces, so we register on both. First PR run uploads to TestPyPI (proving the pipeline); the first push-to-main publishes 0.1.0 to real PyPI.
2. OIDC audience / Trusted Publisher config pointed at github.com/agentculture/cfafi.
3. Environment names: `testpypi` (PR runs) and `pypi` (push-to-main runs) вЂ” both gated on `tests.yml` passing.

No API tokens stored in the repo or GitHub secrets вЂ” Trusted Publishing is mandatory. `docs/SETUP.md` gets a short "for maintainers" section describing how to register the Trusted Publisher on PyPI (one-time, human step outside this PR's code changes).
